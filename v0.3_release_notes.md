# v0.3 bedrock

This release tags the v0.3 Cornerstone U.S. model (2024).
Updates made in bedrock to generate that model and prepare for new features are described below.

Snapshot SHA: `c60bdf4308cb660eee80a246214901cff9122820`
GCS prefix: `gs://cornerstone-default/snapshots/c60bdf4308cb660eee80a246214901cff9122820/`
Canonical config: `2025_usa_cornerstone_v0_3`

## What's Changed

### Core pipeline & configuration

- Canonical pipeline defaults to `2025_usa_cornerstone_v0_3` (2024 IO+GHG, using UMD GHGIA 2024 inventory).
- Prior release stack preserved as `2025_usa_cornerstone_v0_2` for historical comparison.
- Snapshot baseline bumped to `c60bdf4…` (`v0.3.0`); alpha/beta snapshot SHAs registered in `releases.py` and `USAConfig`.
- Industry output `x`: `expand_industry_output_vector` sums many-to-one BEA parents (government-enterprise dollars into electric power `221100` and transit `485000`), correcting inflated D/N where E carried those tons under a first-parent-only expand.
- A-matrix: CEDA fallback scaling with summary A/q dollar-year adjustment (`scale_a_matrix_with_ceda_method_as_fallback`, `adjust_summary_A_and_q_dollar_year`); industry price-index A path removed.
- Inflation: consolidated helpers; single `apply_inflation_to_V` flag; V-norm commodity price index path; cornerstone industry PI gated behind `update_inflation_factors`.
- B matrix: `deflate_x_to_detail_io_year_for_B` replaces `use_useeio_B`; `y_nab` derived from Adom and scaled_q.
- USEEIO baseline config stack (m2 GHG FBS, before-redefinition use tables, B adjustment) for diagnostics comparison.

### GHG emissions

- UMD GHGIA 2024 path (`GHG_national_Cornerstone_2024`) with revised UMD model wiring.
- Cornerstone GHG FBS snapshots extended (2017–2018, 2019–2022, 2024); FBS regen pinned in CI (`cornerstone_ghg_fbs_2024_pin.json`).
- MECS allocation update; `ghgi_mecs` method naming fix.
- Flow mapping files stored in-repo.

### Electricity disaggregation preparation

- Electricity reallocation implemented end-to-end to support disaggregation.
- eGRID plant-level generation, loss, and GHG data; EIA Electric Power Annual FBA to support ongoing electricity disaggregation work.
- Waste-disaggregation matrix export for electricity inputs; disaggregation module rename (phase 1–2) and `cornerstone_disagg_pipeline` refactor.

### EEIO validation & IOT

- q derivation updates and reference comparisons; 2017 margins-after-redef loaders; default `cornerstone_industry_avg_margins`.
- USEEIOR industry CPI when `useeio_margins`.

### Publish & exports

- Phi purchaser-price adjustment in publish path; Phi year panel and dollar-year SEF alignment.
- USEEIO-style Excel writer scaffolding (expanded matrix coverage).
- Supply-chain style CSV export.

### Diagnostics & analysis

- A-matrix time series workflow (steps 2.5–7): cell stability, cross-approach hexbins, impact-weighted sector deep dive, BEA observed A errors, EF diagnostics compile/plot/dispatch, N stability across methods, YoY ECDF panels, scenario-aware EF plots.
- EF diagnostics overlay and multi-panel histogram plotters; dispatch + plot Cursor skills.
- v0.3 release EF diagnostics tooling: sheet registry, `combine_ef_diagnostics` combos (`v0.2_to_v0.3_ceda` / `v0.2_to_v0.3_useeio`), progression / overlay / BLy plotters, optional compare-to FINAL v0.2.
- Wholesale v0→v0.3 waterfall group diagnostics (`v03_waterfall_*`): USEEIO stack (G1 schema/GHG → G2 methods → G3 data → FINAL) and CEDA stack (G1a schema/GHG → G1b waste disagg → G2–FINAL); combine combos, dispatch, and cumulative-plus-sequential plot panels with key-sector N overlays.
- Waterfall Drive sheet IDs pinned to post–industry-`x` redispatches.
- Diagnostics merge cleanup with USEEIO baseline comparison options; inflation on USEEIO diagnostic path; real vs summary derived q plots.
- Snapshot release workflow documented in `bedrock/utils/snapshots/README.md`.

### Infrastructure

- GCS loader skips download when local file exists; snapshot upload path aligned with loader.
- `derive_cornerstone_E` removed; `build_model_manual_check` script for A/q/B from chosen config.
- Google Drive `create_spreadsheet_in_folder` helper; `estimate_suppressed` refactor.

**Full Changelog**: https://github.com/cornerstone-data/bedrock/compare/v0.2...v0.3.0
